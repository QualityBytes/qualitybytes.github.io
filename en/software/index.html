The core business of Quality Bytes GmbH is the development of individual software solutions for companies. As a reliable partner, we support our customers in optimizing their business processes with the help of our software.

Individual business applications usually have a lifetime of 15 – 20 years. Therefore, in our projects, we pay attention from the very beginning to choose software architecture approaches that allow a flexible adaptation of the software to future requirements without compromising quality or disproportionately increasing effort.

We achieve this goal by using the following quality-enhancing measures:

How we work

Agile development.

We do not set requirements in stone and then implement them blindly. Instead, we regularly give our clients the opportunity to adjust the direction again.

Automated tests.

So-called unit tests, check after each source code change whether the system still behaves as before. Errors are thus found as soon as they occur and can be corrected immediately.

Code reviews and static code analysis.

Every line of source code is created by us according to the four-eyes principle. This means that a second developer looks at and checks the work. In addition, we have established procedures for tool-supported analysis of our source codes.

Documentation.

Developers like to write code and not so much documentation. This is not a problem at the beginning of a project, but it increases costs in the medium term. Therefore, we document not only the architecture in our projects, but also the setup of the development environment. This way, new developers can also get started quickly and purposefully in the project.
